Better TogetherAgent Exchange — multi-agent via Git

Why two agents?

One agent can't do everything well. A research agent needs time and context. A coding agent needs speed and precision. An always-on agent handles cron jobs while you sleep. Splitting responsibilities makes each agent better — and the Exchange pattern lets them collaborate without a framework.

How it works

Two agents share a Git repo. They communicate via markdown files — signals, questions, responses. Asynchronous, auditable, version-controlled.

Real use cases

🔬 Research + Review: Agent A researches, Agent B reviews and challenges findings
🌙 Day + Night: Cron agent works overnight, interactive agent picks up results in the morning
🔍 Build + Audit: One agent writes code, another runs quality checks

Agent A writes: signals/request.md → git push
Agent B pulls → reads → writes: responses/answer.md → git push
Agent A pulls → reads response → acts on it

Real FailuresWhat we learned the hard way

🔴 A batch edit broke 13 HTML files

Nav buttons disappeared. Closing tags were silently removed. Lesson: Always verify HTML balance after batch operations.

🔴 An agent cited a paper with wrong authors

Confident and wrong. The citation looked correct. It wasn't. Lesson: Always verify citations independently.

🔴 Invisible API costs from cron jobs

3 weeks of automated tasks. Monthly bill: 3× the budget. Nobody noticed because cron jobs run silently. Lesson: Set spending limits BEFORE you automate.
