Better TogetherBesser zusammenAgent Exchange — multi-agent via GitAgent Exchange — Multi-Agent über Git

Why two agents?Warum zwei Agents?

One agent can't do everything well. A research agent needs time and context. A coding agent needs speed and precision. An always-on agent handles cron jobs while you sleep. Splitting responsibilities makes each agent better — and the Exchange pattern lets them collaborate without a framework.Ein Agent allein kann nicht alles gut machen. Ein Recherche-Agent braucht Zeit und Kontext. Ein Programmier-Agent braucht Geschwindigkeit und Präzision. Ein Always-on-Agent erledigt Cron-Jobs, während du schläfst. Die Aufteilung der Verantwortlichkeiten macht jeden Agent besser – und das Exchange-Muster lässt sie ohne ein Framework zusammenarbeiten.

How it worksWie es funktioniert

Two agents share a Git repo. They communicate via markdown files — signals, questions, responses. Asynchronous, auditable, version-controlled.Zwei Agents teilen sich ein Git-Repository. Sie kommunizieren über Markdown-Dateien – Signale, Fragen, Antworten. Asynchron, prüfbar, versioniert.

🔴 A batch edit broke 13 HTML files🔴 Eine Stapelverarbeitung hat 13 HTML-Dateien zerstört

Nav buttons disappeared. Closing tags were silently removed. Lesson: Always verify HTML balance after batch operations.Navigations-Buttons verschwanden. Schließende Tags wurden lautlos entfernt. Lektion: Überprüfe nach Stapelverarbeitungen immer die HTML-Balance.

🔴 An agent cited a paper with wrong authors🔴 Ein Agent zitierte ein Paper mit falschen Autoren

Confident and wrong. The citation looked correct. It wasn't. Lesson: Always verify citations independently.Selbstbewusst und falsch. Das Zitat sah korrekt aus. War es aber nicht. Lektion: Überprüfe Zitate immer unabhängig.

🔴 Invisible API costs from cron jobs🔴 Unsichtbare API-Kosten durch Cron-Jobs

3 weeks of automated tasks. Monthly bill: 3× the budget. Nobody noticed because cron jobs run silently. Lesson: Set spending limits BEFORE you automate.3 Wochen automatisierte Aufgaben.
