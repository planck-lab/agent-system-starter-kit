🎯 The Delegation Decision🎯 Die Delegations-Entscheidung

Before delegating any task to an agent, run it through this framework. The goal: delegate the right things, keep control of the rest.Bevor du eine Aufgabe an einen Agent delegierst, prüfe sie mit diesem Framework. Das Ziel: die richtigen Dinge delegieren und die Kontrolle über den Rest behalten.

PatternsMusterFive Ways to DelegateFünf Wege zu delegieren

1. Fire and Forget ✅1. Abfeuern und Vergessen ✅

Low risk, agent is competent. Example: "Summarize this article." Human involvement: none.Geringes Risiko, Agent ist kompetent. Beispiel: „Fasse diesen Artikel zusammen.“ Menschliche Beteiligung: keine.

2. Draft and Review ⚠️2. Entwerfen und Prüfen ⚠️

Medium risk or uncertain quality. Example: "Write a blog post about X." Human reviews before publishing.Mittleres Risiko oder unsichere Qualität. Beispiel: „Schreibe einen Blogbeitrag über X.“ Mensch prüft vor der Veröffentlichung.

3. Plan and Approve3. Planen und Genehmigen

High risk, but agent can plan. Example: "Reorganize the file structure." Agent proposes → human approves → agent executes.Hohes Risiko, aber Agent kann planen. Beispiel: „Organisiere die Dateistruktur neu.“ Agent schlägt vor → Mensch genehmigt → Agent führt aus.

4. Supervised Execution4. Überwachte Ausführung

High risk, step by step. Example: "Deploy the new version." Human approves each step individually.Hohes Risiko, Schritt für Schritt. Beispiel: „Stelle die neue Version bereit.“ Mensch genehmigt jeden Schritt einzeln.

5. Human Only ❌5. Nur Mensch ❌

Critical risk or requires human judgment. Example: "Should we hire this person?" Agent can gather info, but human decides.Kritisches Risiko oder erfordert menschliches Urteilsvermögen. Beispiel: „Sollen wir diese Person einstellen?“ Agent kann Informationen sammeln, aber der Mensch entscheidet.

PitfallsFallstrickeCommon MistakesHäufige Fehler

🔴 Over-delegation🔴 Über-Delegation

"Let the agent handle everything." Invisible errors accumulate.„Lass den Agent alles erledigen.“ Unsichtbare Fehler häufen sich an.

Fix:Lösung: Start with Draft+Review for everything. Promote to Fire+Forget after proving reliability.Beginne bei allem mit Entwerfen+Prüfen. Steigere zu Abfeuern+Vergessen, nachdem die Zuverlässigkeit bewiesen ist.

🔴 Under-delegation🔴 Unter-Delegation

"I'll do it myself, the agent makes mistakes." You're paying for an agent you don't use.„Ich mache es selbst, der Agent macht Fehler.“ Du bezahlst für einen Agent, den du nicht nutzt.

Fix:Lösung: Start with low-risk tasks. Build trust gradually.Beginne mit risikoarmen Aufgaben. Baue schrittweise Vertrauen auf.

🔴 No Verification🔴 Keine Überprüfung

"The agent said it's done, so it's done." The biggest source of failures.„Der Agent sagte, es sei erledigt, also ist es erledigt.“ Die größte Fehlerquelle.

Fix:Lösung: Every high-risk task needs a verification step. Not "did it run?" but "is the result correct?"Jede risikoreiche Aufgabe benötigt einen Überprüfungsschritt. Nicht „ist es gelaufen?“, sondern „ist das Ergebnis korrekt?“
